Signal chain

 iPhone/laptop ─(wired DAC)─▶ amp LINE in ─▶ speakers ─▶ room ─▶ built-in mic
      │                                                              │
      └──────────── generates sweep                 records return ──┘
  • Wired output only — never Bluetooth. A2DP re-encodes through a lossy codec that bakes an uncontrolled EQ and time-smear into every trace. Latency is survivable; codec colouration is fatal.
  • Line-level input only (Aux / CD / Tape). Never phono/turntable — it applies RIAA EQ and wrecks the signal.
  • iPhone 15+ : USB-C DAC → 3.5mm-to-RCA → line in. iPhone 14− : Lightning-to-3.5mm → RCA.
  • Mic = the device's own built-in mic, at the listening position (or near the cone for Nearfield / bass work).

Validity & the honesty caveats

  • This is a relative bench, not a calibrated SPL meter. No absolute FR, no SPL, no impedance. Mic error is a magnitude error — so timing/phase and A/B comparisons are immune.
  • Biggest risk: Safari/Chrome may quietly re-apply AGC / noise-suppression / echo-cancellation despite our constraints. Run the Repeatability check — two back-to-back sweeps should overlay within ~1 dB. If they don't, the input is being processed and the numbers can't be trusted.
  • Gate length sets the low-frequency limit: valid above ≈ 1000 / (gate-post ms) Hz. For bass, use Nearfield mode (mic at the cone) to dodge the room.

Mic calibration & using a UMIK

  • The phone/laptop mic is uncalibrated — its error is a magnitude error, so timing, phase, driver-offset and A/B comparisons are already valid. What's NOT valid is the absolute shape of the frequency response.
  • Load a calibration file (miniDSP UMIK .txt, or REW/generic .frd/.cal) on the Measure tab to correct the magnitude shape. With a UMIK-1 plugged into a laptop's USB and selected as the Input device, you get a genuinely calibrated relative frequency response — the kind that drops into VituixCAD.
  • For VituixCAD, the hard part is timing between drivers, not magnitude. A UMIK is a single USB channel, so it can't do a dual-channel loopback timing reference. Use the Driver-Offset mode here to get the acoustic z-offset in mm directly — that's the number VituixCAD asks for and most people just guess. Measure each driver from a fixed mic position and don't move the mic between captures.
  • Absolute SPL is still out of scope. The UMIK's Sens Factor is read and shown, but a browser can't reliably know the input gain chain, so this stays a calibrated-relative bench. For true SPL, use REW on a computer.
